## 名词解释

随着人工智能技术的快速发展，涌现出大量新概念与术语。为达成以下目标：

1. **厘清概念**：系统梳理易混淆的核心术语；
2. **降低沟通成本**：统一定义，减少理解分歧；
3. **规范命名**：建立一致的术语使用标准。

特制定本对照表，供研发、产品及业务团队参考使用。

### 核心术语定义对照表

| 中文名称                   | 英文规范术语                           | 所属层级        | 核心解释                                                                                                   | 存储位置                         |
| :------------------------- | :------------------------------------- | :-------------- | :--------------------------------------------------------------------------------------------------------- | :------------------------------- |
| **会话**                   | **Conversation**<br/>(区分Session)     | 应用 / UI 层    | 用户与 AI 之间一次完整、独立的对话记录。在用户界面中通常表现为左侧列表中的一个条目，是长期记忆的检索单元。 | 数据库 `Conversation` 表         |
| **对话记录 / 消息流**      | **Messages / Chat History**            | 应用 / 数据层   | 某次会话内包含的具体问答列表。在用户界面中表现为右侧聊天面板展示的内容。                                   | 数据库 `messages` 表             |
| **会话上下文（短期记忆）** | **Short-term Memory**（广义 / 应用层） | 应用 / 上下文层 | 从当前对话记录中截取的最近 N 条消息或符合 Token 窗口限制的内容。其目的是使模型记住“刚才聊了什么”。         | 内存（运行时从数据库读取并组装） |
| **Agent 记忆（推理轨迹）** | **Working Memory**（ReAct Trace）      | Agent / 内部层  | Agent 在执行单次任务时，其“思考–行动–观察”循环的记录。包含工具调用参数、返回结果等中间过程信息。           | 内存                             |
| **长期记忆**               | **Long-term Memory**                   | 系统 / 持久层   | 跨会话、跨时间的用户个性化信息，包括偏好、历史事实、知识图谱等。需通过检索增强生成（RAG）等方式唤醒。      | 向量数据库 / 关系数据库          |
